Write some practical JavaScript

Get excited.

We're going to jump right in and display some text on a page.

Getting Started

Copy and paste the following code into a new HTML document: 


				<!DOCTYPE html>
				<html>
				<head>
					<title></title>
					<meta charset="UTF-8">
				</head>
				<body>

				</body>
				</html>

For convenience, you can start with this template when typing the code from the lesson.

Show text on a page

Type the following code in the body of your HTML document and save it:


				<div id="whatever"></div>

				<script>
				document.getElementById('whatever').innerHTML = 'Yes! I win.';
				</script>

Open your HTML document in your browser and, you should see the phrase "Yes! I win." appear on the screen.

Let's break down what's happening here.

  1. First, you've created a div where you want your output to appear, and you've given it an id so you have a way to reference it in your JavaScript.
  2. Next you've written this script to make the text appear in your div: 
    
						document.getElementById('whatever').innerHTML = 'Yes! I win.';

    Here's what that line of script does:

    • In the document, find the element with the id "whatever".
    • Now set the HTML inside that element to show the phrase "Yes! I win."
    • The semicolon indicates the end of the statement.

Show more than just text on a page

innerHTML can have any HTML in it, so it doesn't need to be text only.

Type the following example to see this in action:


				document.getElementById('whatever').innerHTML = '<div style="color:green; background:gold;">Yes! I win.</div>';

Where should you put scripts?

The end of the body is usually the best place to put a script.

Script tags can go anywhere in the head or the body of your page, but if you have a choice, it's usually best to put them as close as possible to the end of the body (just before the closing </body> tag).

That way the rest of the document is loaded so you can reference it, and your page loads faster because scripts aren't blocking the earlier parts.

You can write your JavaScript code directly between the <script> and </script> tags, or you can put your JavaScript in a separate file and call it on your page like this:


				<script src="something.js"></script>

For convenience in these tutorials, we're just going to write our code directly between <script> and </script> so it's all in one place.

Warning: Putting scripts too early

Let's say you want your script to show text in a div.

If you put the script earlier than the div in your document, then you'll get an error.

That's because the script will look for a div that hasn't been created yet.

Warning: Case matters!

JavaScript is case-sensitive.

That means JavaScript treats "hello" and "Hello" as two different words.

If you see me type something like getElementById, you need to keep the same capitalization.

    Try changing the id or the text.

    Next, try creating two different divs (each with a unique id) and putting different text in each.

  2. Type the following piece of code 5 times.

    
						document.getElementById('whatever').innerHTML = 'hi';

    Feel free to look at it, and pay close attention to getting everything right including quotes, capitalization, parentheses, periods, and the semicolon.

  3. Try typing the following code 20 times without looking at any examples.

    
						document.getElementById('whatever').innerHTML = 'hi';

    Even if you feel like you understand this, don't skip it.

    Put on some music if you need to, but take the few extra minutes now to type this out and you'll thank yourself later when everything starts to feel natural.

    If this feels weird, that's okay! You're in good company.

    There's a reason Olympic champions and world-class musicians use repetition on small pieces.

    You're training like a champion now, and that's why you'll make progress faster than other people.

  4. Set the innerHTML of a div to have any HTML you want (not just plain text).

Done? Congrats! You now know how to put stuff on a page with JavaScript.
