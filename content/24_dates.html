The Date object

The JavaScript Date object lets you get and set dates and times.

Type the following to see how it works:


				var d = new Date();
				console.log(d);

In this example, you'll notice that new Date() is used to construct a new Date object.

In JavaScript, a Date object contains a year, a month, a date, a day of the week, an hour, a minute, a second, and a millisecond.

By default new Date() gives you the current date, but you can change it if you want. Here's one way to do that (type this example):


				var d = new Date('march 15 2015');
				console.log(d);

The Date object also contains functions that help you work with dates. You'll see some examples soon.

Get the current date and its parts

The Date object comes built in with some functions that let you get different parts of the date, such as the month or the day.

Type the following code to see how it works:


				var d = new Date();
				console.log( d.getDay() ); // 0 to 6 (0 is Sunday)
				console.log( d.getDate() ); // 1 to 31
				console.log( d.getMonth() ); // 0 to 11 (0 is January)
				console.log( d.getFullYear() );

Warning: getDate starts at 1 but other values start at 0

First I thought that was weird, but it actually makes sense.

It's good that getDay() starts with 0 because that makes it easier to use with an array ['Sunday', 'Monday', 'etc']. Similar story with getMonth() for ['January', 'February', 'etc'].

However getDate() starts with 1 because that way you don't have to add 1 every time you want to say what date it is.

Set the date to 90 days from now

The setDate() function lets you set a date.

You can set the date directly to something like "3" and that will give you the third day of the month.

However, the more useful thing to do is to set the date to X number of days from now. Type the following example to see how it works:


				var d = new Date();
				d.setDate(d.getDate() + 90);
				console.log(d);

There's no need to memorize the above. Just be aware it exists.

Make date math easier

It can be weird to compare dates because you always have to worry about whether you're getting the correct number of days in a given month, whether it's a leap year, what time it is, whether it's morning or evening, what timezone it is, etc, etc, pain, etc, suffering, etc.

Luckily, you can skip all this headache.

JavaScript has a getTime() function that gives you the total amount of time in milliseconds between the current date and January 1, 1970.

Why that date?

It had to do with the limitations of early computers and the size of numbers they could deal with, and now that's the standard zero date.

Anyway, the important part is that getTime() turns a date from a weird object into a convenient number.

Type the following to see how it works:


				var d = new Date();
				console.log( d.getTime() );

If you ever want to do math with dates, getTime() is your friend.
