CSS3 Transitions

In CSS, the transition property determines how an element behaves when its style is changed.

Look at this bit of CSS, for example:


				#mydiv{
					transition: opacity 0.2s;
				}

This CSS says, "Hey, let's look at the element with the id mydiv. Whenever the opacity of that element changes, the transition will take 0.2 seconds."

Type the following code to try it out.


				<style>
				#mydiv{
					transition: opacity 0.2s;
				}
				</style>

				<div id="mydiv">This div will fade in and out.</div>

				<button id="fadeout">Fade Out</button>
				<button id="fadein">Fade In</button>
				
				<script>
				// store the div in a variable so you don't have to keep typing it
				var mydiv = document.getElementById('mydiv');

				// fade out
				document.getElementById('fadeout').onclick = function(){
					mydiv.style.opacity = '0';
				};

				// fade in
				document.getElementById('fadein').onclick = function(){
					mydiv.style.opacity = '1';
				};
				</script>

Here's what the code does:

  1. The CSS transition says that our div will take 0.2 seconds to transition whenever its opacity is changed.

  2. The script says whenever one of the buttons is clicked, change the opacity.

    The CSS transition handles the animation automatically!

There you have it. If you want to animate easily in JavaScript, just throw a CSS transition on there, and you can use JavaScript to change the styles as much as you want!

Tip: Get elements by id more conveniently

If you're going to get an element by its id more than once, just store it in a variable (like in the previous example with mydiv). There are two benefits:

1. You don't have to keep typing that long expression.

2. Your JavaScript will perform better because it won't have to keep searching through the whole document for the same element over and over again.

Transitions work for other properties besides opacity too!

Color change animation

Type the following code to try a color change animation:


				<style>
				#mydiv{
					transition: color 0.2s;
				}
				</style>

				<div id="mydiv">This is the div.</div>

				<button id="turnred">Turn Red</button>
				<button id="turnblue">Turn Blue</button>
				
				<script>

				var mydiv = document.getElementById('mydiv');

				document.getElementById('turnred').onclick = function(){
					mydiv.style.color = 'red';
				};

				document.getElementById('turnblue').onclick = function(){
					mydiv.style.color = 'blue';
				};
				</script>

This works exactly the same way as the previous example except with color instead of opacity.

Movement animation

Now type this one to try some movement. It's the same pattern as the previous examples.


				<style>
				#mydiv{
					display: inline-block;
					background: red;
					padding: 30px;
					transition: transform 0.2s;
				}
				</style>

				<div id="mydiv">This is such a beautiful car.</div>
				<br><br>
				<button id="move100">Move to 100</button>
				<button id="move400">Move to 400</button>

				<script>

				var mydiv = document.getElementById('mydiv');

				document.getElementById('move100').onclick = function(){
					mydiv.style.transform = 'translateX(100px)';
				};

				document.getElementById('move400').onclick = function(){
					mydiv.style.transform = 'translateX(400px)';
				};
				</script>

Quick note: It's easy to forget the way CSS translate is formatted, so just be aware of it to avoid tripping up.

Animate a bunch of properties at the same time

If you want to change a whole bunch of properties at the same time without rewriting the transition for each one, you can just use transition all like in this example (type it out):


				<style>
				#mydiv{
					transition: all 0.2s;
				}
				</style>

				<div id="mydiv">This is the div.</div>

				<button id="blue">Blue</button>
				<button id="redgold">Red and Gold</button>

				<script>

				var mydiv = document.getElementById('mydiv');

				document.getElementById('blue').onclick = function(){
					mydiv.style.color = 'blue';
					mydiv.style.backgroundColor = 'transparent';
					mydiv.style.padding = '0px';
				};

				document.getElementById('redgold').onclick = function(){
					mydiv.style.color = 'red';
					mydiv.style.backgroundColor = 'gold';
					mydiv.style.padding = '40px';
				};
				</script>

Alternatively, you can change the className. Try this example:


				<style>
				#mydiv{
					transition: all 0.2s;
				}

				.highlight{
					background-color: yellow;
					padding: 20px;
				}
				</style>

				<div id="mydiv">This is the div.</div>

				<button id="unhighlight">Unhighlight</button>
				<button id="highlight">Highlight</button>

				<script>

				var mydiv = document.getElementById('mydiv');

				document.getElementById('unhighlight').onclick = function(){
					mydiv.className = '';
				};

				document.getElementById('highlight').onclick = function(){
					mydiv.className = 'highlight';
				};
				</script>

Warning: Remember compatibility

CSS3 transitions will not work in older browsers from IE9 down. In most cases, this shouldn't matter too much as the style will reach the same destination, just without the smooth transition.

To cover certain versions of Safari and the iPhone, you'll need to write CSS transitions like this: 


					-webkit-transition: all 0.2s; /* for safari and iphone */
					transition: all 0.2s;

Other ways to animate:

You already have enough to remember, so don't worry too much about this for now.

I'm only including the following info because I want you to be aware it exists.

If you want more complex CSS3 animations:

In this case you don't have to use transitions at all. Put a pure CSS animation in a CSS class and change the JavaScript className of whatever you want to animate.

I'm not going to go over that here because this is not a CSS course and you have more than enough to remember, but if you're interested, here's a reference on CSS3 animations.

If you want maximum compatibility with older browsers:

You can use something called setInterval to change a CSS property repeatedly at really short intervals to create an animation.

We haven't gotten to setInterval yet, so don't worry about it for now.

If you want fine grained control and high performance animations, and you're not concerned about older browsers:

Then you can use something called requestAnimationFrame.

Don't worry about remembering this. The only reason I'm even mentioning it is so you know it exists.
