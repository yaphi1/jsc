The Event Object - Gain More Control Over Your Events

By now you know how to detect events such as clicks, key presses, and more.

But what if you're not satisfied yet?

What if you want to know where on the page someone clicked, which key they pressed, or where they moved their mouse?

Good news! There's something called the event object that gives you all this info.

What is the event object and how do you find it?

Whenever an event happens (like a click or a key press), JavaScript gives you an object with information about that event.

This is called the event object.

Type the following script to see how it works:


				window.onclick = function(e){
					console.log(e);
				};

Now click anywhere in the browser window and watch the console.

Notice that each time you click, you get a bunch of data about that click. That data is the event object. You can call it whatever you want, but we've just called it "e" because it's short.

Quick Note:
For simplicity, we're checking when the user clicks anywhere on the window, but this could easily be any other event like a button click, a key press inside a text box, or other things.

Get the coordinates of a click

Remember that with objects, you can access their properties. The event object is no different.

Type the following to get the coordinates of each click:


				window.onclick = function(e){
					console.log('x = ' + e.pageX);
					console.log('y = ' + e.pageY);
				};

Neat, right?

There are plenty of ways you can use this.

  • Maybe you have a map and you want to place a marker where the user clicks.
  • Maybe you're creating web analytics software and you want to record where users click most.
  • Maybe you're making a game whose input involves clicking.

Also, feel free to play around with various events and event object properties.

Find out which key was pressed

Type the following to get data about each key press:


				window.onkeydown = function(e){
					console.log(e);
					console.log(e.keyCode);
				};

Here we're looking at two things:

  • The event object (all data about this key press)
  • The pressed key's code number ("a" gives 65, "d" gives 68, and the up arrow gives 38 for example)

When you see the event object in your console, you can click on it to see what properties are available.

Note about keyCode:
There have been a lot of changes in how key presses are handled, so when you're making your big site or app, be sure to check compatibility.

What is a prototype and what does it do?

In JavaScript, objects can inherit properties from other objects.

A prototype is a hidden object that sits in the background and allows other objects to inherit its properties.

All JavaScript objects have a prototype.

It's okay if this sounds a bit confusing.

Try this example to clear things up:


				// your basic car constructor function
				function Car(color, speed){
					this.color = color;
					this.topSpeed = speed;
				}

				// add the showCarInfo function to Car's prototype object
				// this means all the cars will get access to showCarInfo
				Car.prototype.showCarInfo = function(){
					console.log('Color: ' + this.color);
					console.log('Top Speed: '+ this.topSpeed);
				};

				// create a bunch of cars
				var myCar = new Car('red', 200);
				var yourCar = new Car('blue', 150);
				var otherCar = new Car('green', 180);
				var yetAnotherCar = new Car('gold', 240);

				// show your car (notice showCarInfo isn't there)
				console.log('yourCar:');
				console.log(yourCar);

				// try using showCarInfo anyway
				console.log('here is the car info:');
				yourCar.showCarInfo();

What?

showCarInfo() wasn't in the yourCar object, so how were we able to use it?

Here's how:

We changed the Car's prototype object so that all the cars would automatically get access to showCarInfo.

Editing Car's prototype has these advantages:

  • Convenient updates
    You can update the prototype any time and all the cars will get the updates.
  • Less clutter
    All of the cars are referencing one prototype object, so the same showCarInfo function isn't cluttering every single car object.
  • Saves space
    It's better for memory when you're storing a function in one prototype object instead of putting it in every single car object.

The prototype chain

When you call an object's property (a car's color for example), first your browser will check if that property is inside the object.

If it doesn't find the property there in the object, then it'll check the prototype.

Then it checks the prototype's prototype. It keeps going until it finds nothing left.

For example, you might have three kinds of object: Animal, Dog, and Poodle. Poodle inherits properties from Dog, which inherits properties from Animal.

This series of objects in which one inherits from the next is called the prototype chain.

For the sake of maintainability, you don't want to go too many levels deep.

Also, there's no need to worry a whole lot about the prototype chain at the moment. I just want you to see the term at least once.
