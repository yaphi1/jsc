How to avoid getting stuck: Comments, the console, and alerts

Getting Started

Copy and paste the following code into a new HTML document: 


				<!DOCTYPE html>
				<html>
				<head>
					<title></title>
					<meta charset="UTF-8">
				</head>
				<body>


				<script>

				</script>
				</body>
				</html>

For convenience, you can start with this HTML document to type the code from the lesson.

Remember to put all scripts between the script tags.

Comments: Make your code easier to read and debug

Comments are just notes you can put in your JavaScript to remind yourself what's going on. They don't do anything on your page, but they make your code much easier to read later.

Examples of comments:


				// This is a single-line comment.

				/* This is a multi-line comment.
				Put as many lines as you want.
				This is still part of the comment. */

Tip: Commenting is a good habit

When you start to write complex code, it's especially important to comment first.

That way, you won't get stuck as easily because you can do your thinking without being tangled in code.

The console: It can save your life when you have errors

The console is a part of your browser that helps you do three important things (among others):

  • See errors
  • Try out a JavaScript code snippet by typing it there directly
  • Display messages (useful for when you want to test output without showing it on the page)

A quick Google search should show you how to open the console in whatever browser and operating system you're using. (I don't want to waste space here with every combination of browser and operating system.) I recommend memorizing the keyboard shortcut.

To see how the console handles errors, try typing some random nonsense in your script. Open the page in your browser, open the console, and see what happens.

Showing a message in the console

To show a message in the console, put this in your script:


				console.log('this will show up in the console');

Just remember to remove console logs from production websites once they're published so the world won't see your debugging notes.

You can also type JavaScript directly into the console in your browser window.

Alerts: The most basic popups

A JavaScript alert is a small popup with a message. The code looks like this:


				alert('this is an alert');

Try it out and know what it does, but it's not as important as comments and the console.

The only reason I'm including it here is because it's basic and I don't want you to be embarrassed if anyone asks about it.

Exercises:

  1. Write the following code, and then repeat it 3 times without looking: 
    
						alert('hi');
  2. Write a single-line JavaScript comment. Repeat it 5 times without looking. 
    
						// here is a single-line comment
  3. Write a multi-line JavaScript comment. Repeat it 5 times without looking. 
    
						/*
						here is a
						multi-line comment
						*/
  4. Once you're comfortable, write the following code 10 times without looking: 
    
						console.log('hi');
  5. Try entering some JavaScript code into your browser's console just to see what it does.
  6. Use JavaScript to show some text in a div. Add a JavaScript comment of your choice to explain what is happening.
