Create a variable

In JavaScript a variable lets you store information.

Try typing the following example:


				var greeting = 'Hello, my name is Bowser.';
				console.log(greeting);

This code says we're creating a variable called greeting and we're storing some text in it. Then we're showing the text in the console.

The word var tells the browser that you're creating a variable.

Talk the talk

When someone talks about declaring a variable, that just means creating a variable.

Change a variable

Can you change a variable after it's already set?

Yes!

If you want to change the variable after you've declared it, you don't need the var keyword anymore because you've already created the variable.

Here's how to change a variable (type it and see what happens):


				var greeting = 'original greeting';
				console.log(greeting);

				greeting = 'changed greeting';
				console.log(greeting);

Naming your variables

What can you call your variables?

You can call a variable whatever you want, as long as it fits the following rules:

  • YES, the variable's name can have letters, numbers, underscores, or dollar signs
  • NO, the variable's name CAN'T start with a number
  • NO, the name can't be a word that JavaScript already uses to mean something else (examples you've already seen include "var" or "document")
  • Remember that JavaScript is case-sensitive, so "hellothere" counts as a different variable than "helloThere".

What can a variable actually store?

A variable can store a bunch of stuff, but we'll start with strings and numbers.

Strings

A string is just a piece of text enclosed in single or double quotes (either option is fine).

Numbers

A number is... well... a number like 2 or 126 or -55. Notice that it doesn't get wrapped in quotes.

Examples of strings and numbers: (Type the following code in your document, and then use console.log to show each variable.)


				var string1 = 'hello there';
				var string2 = "hi there";
				var a = 1;
				var b = 2;
				var c = a + b;

Dealing with quotes and apostrophes

What if you want to have a string that contains quotes or apostrophes?

Great question.

As long as the type of quote on the outside doesn't match the type on the inside, you don't have to do anything.

For example, these are fine:


				var greeting = "My name's Bowser.";
				var quote = 'I said, "Hi," and they screamed.';

But if you end up using the same type of quote on the inside and the outside, you'll need to do this:

var greeting = 'My name\'s Bowser.';

The backslash tells the browser this character is still part of the text string and not part of the code. This technique is called escaping.

Putting strings together

You can put strings together with a plus sign. That's called concatenating. Here's an example:


				var name = 'Bowser';
				var greeting = 'My name is ' + name;
				var longGreeting = 'Hi there.  My name is ' + name + '. Welcome to my castle.';

Plus means something different for numbers than for strings

Plus adds numbers, but it connects two strings. When you add a number and a string, you get a string. Try out these examples to get a feel for how it all works:


				var a = 2;
				var b = 3;
				console.log(a+b);

				var word = 'hi';
				console.log(word + a);

				var c = '3';
				var d = '4';
				console.log(c+d);

Comments: Make your code easier to read and debug

Comments are just notes you can put in your JavaScript to remind yourself what's going on. They don't do anything on your page, but they make your code much easier to read later.

Examples of comments:


				// This is a single-line comment.

				/* This is a multi-line comment.
				Put as many lines as you want.
				This is still part of the comment. */

Tip: Commenting is a good habit

When you start to write complex code, it's especially important to comment first.

That way, you won't get stuck as easily because you can do your thinking without being tangled in code.

The console: It can save your life when you have errors

The console is a part of your browser that helps you do three important things (among others):

  • See errors
  • Try out a JavaScript code snippet by typing it there directly
  • Display messages (useful for when you want to test output without showing it on the page)

A quick Google search should show you how to open the console in whatever browser and operating system you're using. (I don't want to waste space here with every combination of browser and operating system.) I recommend memorizing the keyboard shortcut.

To see how the console handles errors, try typing some random nonsense in your script. Open the page in your browser, open the console, and see what happens.

Showing a message in the console

To show a message in the console, put this in your script:


				console.log('this will show up in the console');

Just remember to remove console logs from production websites once they're published so the world won't see your debugging notes.

You can also type JavaScript directly into the console in your browser window.

Alerts: The most basic popups

A JavaScript alert is a small popup with a message. The code looks like this:


				alert('this is an alert');

Try it out and know what it does, but it's not as important as comments and the console.

The only reason I'm including it here is because it's basic and I don't want you to be embarrassed if anyone asks about it.
