What is a JavaScript object?

In real life, an object is something like a shirt, for example. That object can have properties. For example the shirt could be red in color and medium in size.

Similarly, a JavaScript object is a thing that can have properties.

More formally, a JavaScript object is a list of name : value pairs separated by commas and enclosed in curly braces.

Type the following example to see how it works:

var shirt = {size: 'medium', color: 'red', sleeveLength: 6}; console.log( shirt.size ); console.log( shirt['size'] );

The code example above first creates an object called shirt. Then then it shows the size property in the console.

If you notice, there are two ways to get the property of an object:

1. object.property 2. object['property']

Most of the time, you'll want to use the first way of getting a property, but you'll want to use the second way if you need to use a variable as the property name.

Type this example to see how it works:

var propertyWeWant = 'color'; var shirt = {color: 'red', size: 'medium'}; console.log( shirt[propertyWeWant] );

Benefits of objects

1. Objects are a clear and organized way to store information. 2. Objects allow you to take up fewer variable names.

With an object: Only one variable gets used.

var settings = {size: 'big', color: 'orange', version:2, etc: 'whatever'};

Without an object: Four variables get used.

var size = 'big'; var color = 'orange'; var version = 2; var etc = 'whatever';

Methods: You can have functions in your objects

A function inside an object is called a method.

Type out the following example to see a method in action:

var bird = { color: 'blue', face: 'so majestic', fly: function(){ console.log("I'm flying!"); } }; bird.fly();

Try this example to get some more practice with methods:

var person = { name: 'Luigi', greet: function(otherperson){ console.log('Hello ' + otherperson); }, favoriteColor: 'green' }; person.greet('Bowser'); Warning:

Don't forget the commas in an object.

The Document Object Model (DOM) and a pleasant surprise

Surprise! You've actually been using objects all along!

Every time you typed document.getElementById('whatever'), it turns out that the document is an object, and getElementById is a method.

When people refer to an HTML document as the DOM or the Document Object Model, that's because your browser treats the document as an object.

That's right! The document is an object!
