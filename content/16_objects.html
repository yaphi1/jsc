What is a JavaScript object?

An object is a convenient way of storing data.

It's almost like an array, but the brackets look like this { } and each item in an object has a name.

An array looks like this:


				var arr = ['thing 1', 'thing 2'];

But an object looks like this:


				var obj = {name1:'thing 1', name2:'thing 2'};

Object properties

In real life, an object is something like a shirt, for example. That object can have properties. For example the shirt could be red in color and medium in size.

Similarly, a JavaScript object is a thing that can have properties.

Type the following example to see how it works:


				var shirt = {size: 'medium', color: 'red', sleeveLength: 6};

				console.log( shirt.size );
				console.log( shirt['size'] );

The code example above first creates an object called shirt with three properties: size, color, and sleeveLength.

Then then the example shows the size property twice in the console.

If you notice, there are two ways to get the property of an object:

  • Dot notation: 
    object.property
  • Bracket notation: 
    object['property']

Most of the time, you'll want to use the dot notation because it's faster and more readable.
Hint: Don't worry about remembering which one to use. Just use dot notation, and if you come to a situation where you can't use it, you'll know. Trust me.

You'll want to use the bracket notation if you need to use a variable or non-standard text characters in the property name.

Type this example to see how bracket notation can be useful:


				var language = 'English';
				var greeting = {English: 'hi', Spanish: 'hola'};

				console.log( greeting[language] );

This is convenient because you can change the desired language and get the correct greeting.

As you work with more data, you'll be able to appreciate using an object like this instead of having a long set of if statements (if English ... , if Spanish ... , if French ... , etc).

What is an object key?

If you hear anyone talking about keys in an object, a key is just another word for a property name. 


					var obj = { key : 'value' };

Now you know what it means, so you won't get confused if you hear it.

Benefits of objects

  • Objects are a clear and organized way to store information.
  • Objects allow you to use up fewer variable names.

With an object: Only one variable gets used.


				var settings = {size:'big', version:2, color:'orange'};

Without an object: Three variables get used.


				var size = 'big';
				var version = 2;
				var color = 'orange';

Methods: You can have functions in your objects

A function inside an object is called a method.

Type out the following example to see a method in action:


				var bird = {
					color: 'blue',
					face: 'so majestic',
					fly: function(){
						console.log("I'm flying!");
					}
				};

				bird.fly();

Try this example to get some more practice with methods:


				var person = {
					name: 'Luigi',
					greet: function(otherperson){
						console.log('Hello ' + otherperson);
					},
					favoriteColor: 'green'
				};

				person.greet('Bowser');

Warning: Don't forget the commas

Now that your objects are getting more complex, it's a good time to remind you not to forget the commas between items.

This is such a common mistake.

The Document Object Model (DOM) and a pleasant surprise

Surprise! You've actually been using objects all along!

Every time you typed document.getElementById('whatever'), it turns out that the document is an object, and getElementById is a method.

When people refer to an HTML document as the DOM or the Document Object Model, that's because your browser treats the document as an object.

That's right! The document is an object!
